# 1장. 생성형 AI가 열어갈 세상
## 영화 "인턴", 그리고 챗GPT
- 학습 메커니즘에서는 **`피드백`** 이 중요
- "인턴"에서는 30대 CEO와 70대 인턴이 서로 피드백을 주면서 학습
- 학습 피드백의 포인트는 명령이 아닌 **`서로 대화를 맞추는 것`**
- 길게 어떤 것을 요구하는 것이 아니라 있는 **내용들을 한 가지씩 잘 얘기해 주는 것**이 좋음 

## 충격의 일주일
- 2023년 3월 14일 GPT-4가 공개된 후 "충격적인 AI의 위력"에 놀람
  - 처음에는 많은 업무를 처리해줄 것으로 반겼으나, 점차 직업을 잃을 수 있다는 우려의 목소리가 높아짐

## 생성형 AI를 똑똑하게 활용하려면
- ChatGPT의 의미: Chat(대화형) + Generative(생성형) + Pretrained(사전학습한) + Transformet(다음 단어 예측 모델)

### 챗(Chat), 사람을 불러모은다는 것! 
- 알파고는 바둑계에는 큰 변화를 일으켰지만, 일반인들에게 큰 영향을 주지 못함
- 반면 챗GPT는 2022년 11월 30일 공개 5일만에 100만명, 두 달만에 1억명의 회원을 모았음 -(why?)-> **`대화하는 인공지능`**
- 챗GPT는 찰떡같이 **`대답을 한다`**

### Generative/Pretrained, 생성형은 검색이 아니다 
- Generative은 뭔가 만들어 낸다는 것, Pretrained은 기존에 있었던 것을 미리 사전학습했다는 것 --> **`검색이 아님`**
- 기존에 사람과 AI가 하던 역할이 바뀜 (단순 정보 표시 vs. 판단/구분)

### 챗GPT가 문장을 만드는 방법, 트랜스포머 
- 트랜스포터 = 빈칸채우기
  - 단, 미리 공부한 내용을 바탕으로 빈칸을 채움 -> **`통계학적 앵무새`**


### 생성형 AI는 통계학적 앵무새? 인간 피드백 강화학습
- 문장을 통째로 외워 버리는 강화학습 모델
- **`인간 피드백 강화학습`**: 특정 영역에 대해서 정확한 결과를 전달할 수 있게 사람이 보완해 주는 방식

## AI 시대, 기업이 할 수 있는 일들

### 세상을 뒤흔든 챗GPT, 비즈니스의 게임 체인저 
- 챗GPT는 의사 및 변호사 같은 전문직 자격시험을 통과하는 등 지적능력 여러차례 입증
- 지식산업의 상당 부분을 대체할 가능성을 배제하기 어려움 -> **`게임 체인저`**
- 콘텐츠 산업(교육, 광고, 메타버스), IT산업(반도체, 데이터 보안)을 중심으로 큰 영향을 줄것

### 기업에서 생성형 AI를 도입하는 방식 
- 생성형 AI를 그대로 쓰지 않고, API형태로 가져와서 쓰는 경우 많음
  - why? 보안, 해킹, 개인정보, 비용, 데이터 활용 관련 소송 등

- 기업의 생성형 AI 도입 방식 3종
  1) 우리 회사만을 위한 AI 모델을 만드는 방식
     - 사전학습 부터 모든 가중치를 조절
     - 맞춤형으로 자유도가 높지만 비용 높음
  2) 파인튜닝 방식
     - 만들어진 모델에 추가로 미세조정을 해서 좀 더 섬세하게 쓸 수 있게하는 것
     - 거대모델이 관심업무 분야에 답변을 잘 못할 때, 우리 분야레 대한 데이터를 추가 학습시켜 잡변을 잘하게 만드는 것 
  3) 프롬프트 엔지니어링만하는 방식
     - 생성형 AI에게 답변을 잘 받을 수 있도록 유도하는 기법

### 모건스탠리, 블룸버그GPT의 파인튜닝 
- 모건 스탠리는 23년 3월 GPT-4가 나왔을 때 OpenAI와 파트너쉽을 맺고 회사 내에 있는 데이터들을 다 학습 시킴
  - 직원들의 수작업 줄이고, 복잡한 문서 빨리 분석, 효율성 향상, 시간과 자원을 더 가치있는 업무에 집중

### API 비즈니스, 원래의 서비스에 더 똑똑한 비서 탑재 
- 여행 전문 앱에 대화형 AI를 붙이면 여행 코스를 잘 짜줌

### 생성형 AI가 쏘아올린 금융권 AI의 미래, AI 뱅커 
- 일부 은행은 자체 언어모델을 만들거나 여러 언어 모델을 API형태로 가져와 비지니스를 구상중
- AI 휴먼이 탑재된 은행 앱 개발 완료 -> 보안, 법적인 문제?

### AI 모델 & 나만을 위한 패션 디자이너 
- 판매량, 평점, 댓글 기반으로 옷 추천, 코디, 신규 디자인 요청 
- 소통형 디지이너, 코디 역할 가능

### 생성형 AI가 만드는 새로운 소통과 비즈니스 
- 컴퓨터에 일을 시키기위해서는 코딩 필요 -> 이젠 언어로 말하고 언어로 답변 받음
- 일의 **`본질`**을 잘 아는 사람이 주목받을 것

### 스마트 팩토리에 적용된다면? 
- 공장 문제 발생시 말만하면 생성형 AI가 알아듣고 고쳐줄 것
- 사용자가 지시하면 스스로 드론을 조종하는 코드를 생성해서 제어하는 사례 등장 [MS 시연 영상](https://www.youtube.com/watch?v=i5wZJFb4dyA&t=87s&ab_channel=MicrosoftAutonomousSystems%26RoboticsResearch)
- 구글의 PaLM-2 모델 장착 로봇은 "물을 쏟앗는데 물 닦기 좋은 것 가져다줘" 하면 주변의 스펀지를 인지해서 가져다줌 (잘 잡지는 못함) 
- 구글의 시각-언어-행동(VLA) 모델 탑재 로봇은 별도의 학습없이도 스스로 인터넷 정보를 가지고 학습해서 판단하는 수준에 이름


### 내 휴대폰 안의 대화형 AI, 라마-2 
- 라마는 언어모델의 소스코드를 공개 -> 연구자들이 다양하게 계속 고쳐옴

### 2군의 강력한 추격 속도 
- 앤트로픽의 베드락 플랫폼 (아마존 투자), 클로바X


### 전략적 방향성이 필요하다 
- 어느 모델이 우리 회사에 맞는지? 어떤 데이터로 학습할지, 성능 평가는 어떻게? 문제발생시 대응은?
- 편향성을 어떻게 해결할지? 

### 모바일과 생성형 AI 시장, 어떻게 움직일까?
- 사용성을 어떻게 높일 것인가?
- 기업 특화 서비스는 무엇인가?
- 효율과 비용 최적화 -> 오픈소스 기반 파트너십 늘어남
- 용어가 아닌 "기술이 스며드는 시대"
- "배달의 민족", "카카오톡" 처럼 사용자들의 습관을 바꾸고 생활 속에 깊숙이 파고들 수 있는 서비스 만드는 곳이 변화 주도 

## 2024년, AI의 실체를 마주하는 원년

### AI 비즈니스가 본격화된다 
- AI 에이전트
  - ex) 챗데브 - AI와 채팅을 통해 게임을 개발. CEO, 프로그래머, 디자이나 역할의 AI두고 협업해서 게임을 만들라고 시켰더니 괜찮은 결과물 만들어냄
  - ex) 오토GPT, 에이전트 GPT

### 아이폰 모먼트와 유사한 ‘생성형 AI 모먼트’ 
- 우리가 의식하지 못하는 사이에 생성형 AI는 일상속에 자연스럽게 녹아들 것
- '생성형 AI'라는 이름으로 굳이 부르지 않을 때 기술의 진보가 이루어질 것


### 맞춤형 거대언어모델(LLM)이 온다 
- 기업들은 자체적으로 보안을 지킬 수 있는 맞춤형 거대언어모델을 만들려고 하는 추세 -> 라마2를 오픈소스로 공개되었기 때문에 가능


### 개인이 기업이 된다 
- 스마트한 도우미가 되어 업무 효율성 강화
- 24년은 AI 도구 잘 활용하는 법을 아는 것이 중요

### 법적, 윤리적 쟁점을 극복할 수 있을까? 
- 데이터 사용 권리와 돈 문제들이 복잡해짐
- 생성형 AI가 만든 결과물의 저작권이나 상업적 사용, 악용 가능성에 대한 고민 필요
- 만들어진 이미지, 보이스로 일반인들에 대한 해킹이나 직접적인 피해 가능성
- 표절 판단도 어려움

### 챗봇 네이티브를 준비하라! 
- `챗봇 네이티브` 세대는 AI와 대화하며 다양한 생각을 표현하고 논의할 수 있는 기술을 습득

- 챗봇 사용에 대한 가이드라인이 나올 것

## 코파일럿이 바꿀 업무 환경

  - 예측과 분석의 이유까지 설명
- 코파일럿으로 PPT 파일을 단숨에 생성
  - 아까 만든 워드파일을 기반으로 PPT 만듦

## 윈도우에 코파일럿이 들어간다고?
- PC 세팅 변경 (집중 모드, 다크 모드)
- 문서 요약
- 음악추천 


## 코딩도 뚝딱, 깃허브 코파일럿
### 월 1만원 정도에 최고의 코딩 선생님을 옆에
- 깃허브 코파일럿은 주석을 쓰면 자동으로 소스 코드를 작성
  - ex)"tetris 게임을 pygame이라는 라이브러리를 이용해서 구현한다."

### 네이버의 새로운 언어모델 하이퍼클로바X
- 네이버가 23년 8월 24일 일반 공개
- 한국어 데이터 5600억 토큰: 뉴스 50년치, 네이버 블로그 9년치 데이터량, GPT-3 대비 6500배
- 자사의 뉴스, 블로그, 카페 등의 데이터 이용하여 저작권 이슈 자유로움

### 클로바X
- GPT-4의 플러그인처럼 **스킬** 이라고 하는 기능 제공 -> 현재는 여행, 네이버 쇼핑처럼 안에 정보를 검색해서 가져오고 있으며, 향후 증가될 것

### 사용자의 간지러운 부분을 긁어주는 AI 검색, 네이버 큐(cue)
- 네이버가 23년 9월 20일 공개
- 사용자가 자연스럽게 질문하면 큐는 이를 이해하고 원하는 정보를 검색하여 제공
- 뛰어난 AI 기반 검색 기능: 네이버 지식인, 이미지 서비스와 연동 

### 구글, 반격의 시작?_바드와 연결되기 시작한 구글 앱 서비스
- 23년 2월 급하게 바드를 출시했으나, 시연회 결과 처참해 하루만에 주식 7.7% 급락함
- 코닥의 딜레마? 검색 기반 광고시장을 위협할 생성형 AI 기술을 키우는데 의지 부족 의심 ->현재는 AI에 올인하는 듯
- 구글 바드의 익스텐션(GPT의 플러그인과 유사)을 통해 유튜브, 구글 워크스페이스, 구글 항공편, 구글 지도 서스 통합중

### 진정한 멀티모달 서비스로 진화하는 챗GPT
- 23년 9월 25일 음성으로 질문과 답변을 주고 받을 수 있는 기능, 이미지를 보고 답하는 기능을 추가함
- 유료 서비스인 쳇GPT 플러스와 챗GPT 엔터프라이즈에서 해당 기능 가능
- 그래프(이미지)를 올리고 분석하는 것도 가능

### 메타버스에서 만나는 생성형 AI
- 메타버스는 생성형 AI를 통해 더 몰입도 있게 변화됨
  - ex) 말을 걸어봤을 때 생명력 있는 답변이 가능한 NPC
- 엔비디아 에이스(Nvidia ACE): 게임 캐릭터들이 자동으로 대화할 수 있게 해줌

# 4장. 프롬프트 엔지니어링 한눈에 보기
## 좋은 질문을 하는 8가지 방법
### 프롬프트, 질문이 바뀌면 결과가 바뀐다
- 8세 천재 아이처럼 다루자
- 지시를 구체적으로 해야 한다
- 명확한 단어를 사용해야 한다
- 질문의 맥락이나 배경정보를 주자 - 일종의 사전학습
- 좋아하는 국어 선생님 대하듯 올바른 형식을 갖추자
- 질문이 구체적으로 안 떠오르면 개방형 질문
  - ex) "내가 OOO에 대해 알고 싶은데, 너한테 어떻게 질문하면 좋을까?
- 챗GPT는 사건을 해결하려는 탐정처럼 대답 -> 원하는 대답이 나오도록 추가 정보나 맥락을 제공하면 효과적
- **챗GPT의 답변은 검증이 필요하다** 

### 구조화를 위한 프롬프트 엔지니어링
1) 주제: 구체적인 내용, 범위한정
2) 옵션: 문체(개조식/서술식, 전문적/쉽게), 언어(한글, 영어)
3) 양: 글자수, 단어수, x개 제한
4) 포맷: 텍스트, 리스트, 트리, 표, 코드
5) 방법: 생성, 요약, 추출, 카테고리화, 그룹
   

### 퓨 샷(Few Shot)을 줘라!
- 예시 Shot을 많이 줄수록 챗GPT의 대답에서 다양성과 창의성이 높아짐
