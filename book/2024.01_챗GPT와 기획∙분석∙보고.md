# [1부] ChatGPT로 기획∙분석∙보고하기
 
## 1장: 초안을 만드는 ChatGPT
### 기획 초안, 분석 초안, 보고 초안 만들기가 제일 어렵다
 - **기획**은 문제를 파악하고 조사를 하고 아이디어를 내고 전략을 수립하고 과제를 도출하는 일
 - **분석**은 데이터를 정제하고 구분하고 분유해서 연관된 데이터를 찾고 패턴을 찾고 원일을 찾아 통찰하고 시사하는 일
 - **보고**는 기획하고 분석해서 만든 안을 상사나 고객이 편안하게 읽고 헤석할 수 있게 일목요연하게 만드는 일

### ChatGPT가 초안을 만드는 시대가 왔다
- 무엇을 기획하고 분석하고 보고해야 하는지 물어보자.
  - ex) 회사에서 새 만두제품을 개발하여 출시해야 한다. 무엇을 기획하고 분석하고 보고해야 할 지 알려줘.


### ChatGPT가 기획, 분석, 보고를 안내한다
 - 우리가 어떻게 일을 하면 되는지 아내
 - 더 좋은 안을 찾아 제시
 - 잘못한 것이 있는지 검토

# [2부] ChatGPT로 기획하기
 
## 3장: ChatGPT와 문제 파악
### 문제 인지, 문제 인식, 문제 의식
- 우리는 늘 바쁘다는 핑계로 개선할 생각을 못함 
  - 오른쪽 사람은 문제가 있다는 정도는 암 -> 바쁘게 일함
  - 중간 사람은 문제가 무엇인지(네모 바퀴)는 인식하나 해결의지 없음
  - 왼쪽 사람은 해결책을 제시하나 당사자가 받아 들이지 않아 멀뚱히 쳐다봄

- **문제 인지**: 문제가 있다 없다를 판별하는 능력
- **문제 인식**: 문제의 특성을 판단해서 어떤 문제인지 알아내는 능력
  - 문제의 3종류
    |문제의 종류|세부문제|설명|
    |:---:|:---:|:---|
    |발생형 문제|미달 문제|정해진 목표나 과제를 달성하는 못한 문제|
    |          |일탈 문제|정해진 기준이나 규칙에서 벗어난 문제|
    |탐색형 문제|개선 문제|부족하거나 잘못된 문제|
    |          |강화 문제|효율을 높일 수 있는 문제|
    |설정형 문제|개척 문제|미래 사업과 관련한 문제|
    |          |대비 문제|미래의 위험을 최소화 하는 문제|

  - 문제 인식력이 높으면 문제를 잘 분간하고 대처
    - 발생형 문제는 서둘러 조치
    - 탐색형 문제는 AS-IS, To-BE를 비교
    - 설정형 문제는 트렌드를 조사하고 전략을 기획

- **문제 의식**: 문제를 해결하고자 하는 의지가 포함
  - 문제를 인지, 인식하면 누구나 문제를 해결할 것 같지만, 실제로는 그렇지 않음
  - 조식에서는 자기가 맡은 책임과 권한 안에서만 문제 의식이 생김

### ChatGPT로 문제를 인지하고 인식하기
- 업무와 관련된 정보를 빠르게 얻어, 업무관련 발생형 문제 파악
  - ex) IT회사에서 생성형AI를 활용하여 업무를 처리할 때 발생할 수 있는 문제를 모두 나열해줘.
- 탐색형 문제 접근
  - ex) IT회사에서 생성형AI를 활용하여 업무 효율을 높일 수 있는 방법을 7개 알려줘.
- 설정형 문제 접근
  - ex) 향후 IT회사에서 생성형AI의 발전에 대비해야 할 것을 5개 알려줘.

### 문제를 제대로 파악했는지 아는 방법
- 문제의 종류별 보고서 용어
    |문제의 종류|문제 상황|목적|기대효과|파급효과|
    |:---:|:---:|:---:|:---:|:---:|
    |발생형 문제|장애, 사고|정상화, 수습|재발 방지|신뢰 향상|
    |탐색형 문제|부족, 악화|보충, 개선|효율 향상|수준 제고|
    |설정형 문제|포부, 예상|준비, 대비|지속 가능|기회 창출|


### 문제를 균형 있게 보고하기
- 특정 문제만 자주 보고할 경우 개인의 특성으로 평가받을 수 있음
  - ex) 발생형 문제를 주로 보고하면 늘 사건사고만 보고하는 직원으로 생각 
- 낮은 직급의 직원도 발생형 문제를 넘어 탐색형 문제와 설정형 문제에 관심을 두면 좋음
- 반대로 직급 높은 사람도 설정형 문제를 넘어 탐색형 문제와 발생형 문제에 관심을 둘 것


### 문제와 문제점
- 문제: 설비 고장, 매출 하락, 불안한 미래같이 현상으로 나타남
- 문제점: 문제를 일으키는 다양한 요인 -> '문제를 파악한다'는 것은 문제점을 파악하는 것
- 문제점 파악
 - 많은 사람이 문제점의 일부만 찾거나, 편향된 분야의 문제점을 찾는 등 찾는 문제점을 다 고려하기 어려움
 - ex) 전구가 깜빡이는 요인에는 어떤 것이 있나? <답변 후 재질의> 확인해 봐야 할 요인 10개 더 말해줘.  
- 프레임워크를 통한 문제점 파악
 - ex) 3C 분석 및 SWOT 분석으로 설명해줘 


## 4장: ChatGPT와 시장 조사
### 보통 직장인의 조사 착각 3가지
- 뭔가 엄청난 것을 조사한다
  - ex) 본업이 `ESG 보고서 작성 컨설팅`인데 `ESG 시장 규모`를 조사
  - 사업 영역에 대한 선을 제대로 긋지 못하기 때문
  - 시장 조사는 **표준산업분류**로 하는 것이 좋음
    - 산업분류는 IPO(Input, Process, Output)를 기반으로 투입물, 생산물, 생산활동이 유사하면 같은 업종으로 분류
    - 업종은 업태(업의 형태로 투입과 활동으로 구분)와 종목(산출물의 종류, 산출물 전달방식으로 구분)
      - 업태 ex) 농업, 광업, 교육 서비스업, 전문 과학 및 기술 서비스업 등
      - 종목 ex) 경영 컨설팅업
  - 팀장은 조사 지시시 업종, 조사의 범위/범주 등을 명확히 알려줘야 함
- 자기가 처음 조사한다
  - 누군가가 먼저 조사했다는 생각을 가지고 시작하는 것이 중요 -> 선행 조사정보/사람을 활용
- 자기가 직접 조사한다
  - 우리팀이 할 수 있는지 사전에 철저히 검토
  - 자료의 가치가 높을수록 직접조사하는 것이 문제가 될 수 있음 (ex: 여론조사) -> 전문업체 외주
  - 어떤 문제에 대해 기초적인 지식과 논리적인 추론으로 대략적인 근사치를 추정할 수도 있음
    - ex) 페르미 추정
 
## 5장: ChatGPT와 아이디어 기획

### 아이디어의 원천
- 챗GPT도 과거의 자료를 수집하고 학습하고 분석해서 아이디어를 얻음
- 비지니스 아이디어는 분석에 기반해서 나와야 함
- 외부 환경 분석과 내부 환경 분석에서 아이디어를 찾는 것이 자연스러운 수순
- 고객에게 직접 묻거나 로드맵을 참고하는 것도 좋은 방법

### ChatGPT의 브레인스토밍
- 마구잡이로 물어보기
  - ex) 이번에 출간할 책이 재미있는 책입니다. 어떤 제목이 있을까요? <답변 후> 제목이 밋밋합니다. 과감한 제목을 알려주세요. <답변 후> 재치 있는 제목을 알려 줘.

### 아이디어가 아니라 아이디어 설명
- 머릿속에 아이디어는 많다. 그러나 아이디어를 설명하기가 어렵다. -> 설명을 제대로 하지 못하기에 아이디어를 내지 않음
- 아이디어가 있는 것과 아이디어를 설명하는 것은 완전히 다른 능력
  - 창의력은 아이디어를 떠올리는 능력 
  - 아이디어를 설명하는 능력은 어떻게 기르나?

### ChatGPT로 아이디어 설명하기
- 챗GPT에게 아이디어를 알려주고 설명하는 글을 써 달라고 하면 됨
  - ex) 내 책의 제목을 "영웅"으로 정했어. 왜 이 제목을 선택했는지 설명하는 글을 200자로 써줘.
- 내 아이디어에 대한 예상 질문을 달라고 할 수 있음
  - ex) 종이 빨대 아이디어를 설명하려고 해. 이 아이디어의 설명을 듣고 예상하는 질문을 5개 줘.

### 아이디어 설명 대신 질문하기
- Why 에 대한 질문을 챗GPT에 물어보고 초안으로 사용

### 사내 투자를 받는 6가지 질문
- Why we? 왜 우리 회사가 해야하는가?
  - 기업의 정체성(Identity)에 대한 설명
- Why now? 왜 하필 지금 해야 하는가?
  - 비지니스 타이밍 (ex: 에어비앤비(서브프라임 모기지 사태), ZOOM(코로나 펜데믹))
    - 외부 타이밍: 시장이나 고객이 우리 제품/비지니스를 받아들일 준비가 되었느냐
    - 내부 타이밍: 우리가 제품/서비스를 제공할 준비가 되어 있느냐
- Why don't they? 다른 회사는 왜 안하는가?
  - 알고 설명하기 매우 어려움 (자료가 없으니까)
  - 다른 회사는 왜 안하는지 챗GPT에게 물어보기 가능
- If we don't? 우리가 안하면 어떻게 되는가?
  - 최종훈 교수의 인생교훈
    ```
    갈까 말까 할 때는 가라
    살까 말까 할때는 사지 마라
    말할까 말까 할 때는 말하지 말라
    줄가 말까 할 때는 줘라
    먹을까 말까 할 때는 먹지 마라 
    ```
  - 사내 투자심의위원: **투자를 할까 말까 할 때는 하지 마라.**
    - 이는 투자에 대한 책임이 따르기 때문 -> 투자 안하면 책임 없음
  - ex) 우리가 디지털 전환을 안하면 어떻게 되는가? 200자로 알려줘.
- Can we? 우리가 할 수 있는가?
  - 역략 얘기가 아님 -> 의지에 대한 얘기
  - 기획을 진행하면서 역량(내외부 역량 동원)과 의지가 높아질 수 있음
  - ex) 우리가 할 수 있는가? 200자로 알려줘.
- Who lead? 누가 맡아서 해야 하는가?
  - 기획자가 책임자 후보를 제시(본인이나 다른 적임자)
  - ex) 누가 맡아서 해야 하는가? 200자로 알려줘.


## 6장: ChatGPT와 전략 수립
### 기업의 전략은 신년사에 다 있다
- 보통 매년 연말이 되면 다음 해 전략을 수립 -> 사내 비밀문서
- 신년사에 경영 전략이 나타남

|구분|내용|
|:---:|:---|
|인사|	새해의 시작을 맞아 모든 임직원에게 감사의 말씀을 전하며, 수고에 대한 인정 표시
|외부| 환경 리뷰	글로벌 시장 동향과 산업 변화에 대한 주목 및 대응 전략에 대한 강조
|내부| 경영 리뷰	지난 해의 도전과 성취를 감사하며, 내부 프로세스 향상과 협업 강화에 대한 성과 강조
|새해 전망|	혁신적이고 지속 가능한 비즈니스 모델 추구, 시장 기회 포착과 새로운 파트너십 구축 계획 강조
|비지니스 제약 설명|	전망을 기반으로 돈, 사람, 시간의 효율적인 관리, 비즈니스 제약에 대한 적극적 대응을 모색하는 의지 표현
|경영 전략 설명|	디지털 전환과 지속 가능성에 중점을 두는 경영 전략 강조, 혁신과 효율성을 동시에 추구하는 방향성 표현
|마지막 인사|	모든 임직원에게 마지막으로 새해 복 많이 받으라는 인사와 함께, 그들의 기여에 대한 감사의 마음을 전하는 마무리 부분

### 팀 전략 만들기
- 팀 전략도 경영 전략 수립 방법과 그 산출물을 그대로 이어받아야 함- 

### ChatGPT로 팀 신년사 만들기
- 팀 전략을 구상했으면, 팀 신년사 배포 추천
  - ex) 팀에서 외부 환경을 리뷰한 결과 '물류 롼경에서 구조적 변화 가속화'가 핵심 메시지로 도출되었다. 이 메시지의 세부 내용을 발표형식을 200자 써줘.
- 내부 경영 리뷰
  - ex) 우리 팀은 작년에 어려운 물류 환경에서도 물류 경쟁력 강화를 이뤄냈다. 이 내용을 구체화하여 발표 형식으로 200자 써 줘.
- 새해 전망을 구체화
  - ex) 새해에는 수출과 소비 악화로 물류 산업이 어려워진다는 내용의 전망을 발표 형식으로 200자 써 줘
- 비지니스 제약 3가지, 물류 가격 정체, 최고 물류 관리 인재 부족, 급변하는 물류 환경에 대해 구체적으로 작성 요청
  - ex) 우리 회사는 물류 기업이다. 다음과 같은 3가지 비지니스 제약에 대해 하나씩 100글자 이내로 구체화 해서 발표 형식으로 써줘. 1) 물류 가격 정체, 2) 최고 물류 관리 인재 부족, 3) 급변하는 물류 환경
- 팀의 전략 수립
  - ex) 우리 팀의 올해 전략은 다음 3가지다. 각각의 전략을 100자로 구체화해 줘. 1) 물류 혁신 성장, 2) 최고 물류 관리 인재 확보와 육성, 3) 핵심 제품 물류 경쟁력 확보

### 전략과 방안과 과제
- 돈, 사람, 시간의 `제약`으로 인해 `전략`이 요구됨
- `방안`은 전략을 실행하는 기준이나 방법
- 방안중에서 추진하기로 선택한 것이 `과제`
- 전략을 구체화하면 과제가 되므로 올바른 과제를 선정하고 추진 -> `기획`


### 전략과 과제 매트릭스
- 전략과 과제를 안다면 -> 여러 과제중 우선순위 결정
- 전략을 알지만, 과제를 모른다면 -> 과제를 기획
- 전략을 모르는데 과제만 안다면 -> 전략과 정렬되게 검토
- 전략, 과제 모두 모름 -> 일단 전략부터 수립

### 데이터 정리와 데이터 분석
- 분석: (사전적 의미) 칼로 자르고 도끼로 벤다는 뜻
  - 얽혀 있거나 복잡한 것을 풀어서 개별적인 요소나 성질로 나누고 나서 -> 데이터 정리 
  - 논리적으로 해명하는 것 -> 분석
- 정량 데이터 분석: 복잡한 것을 풀어서 개별적인 요소나 성질로 나누ㅜ어 논리적으로 해명
- 정성 데이터 분석: 개념이나 문장 보다 단순한 개념이나 문장으로 나누어 그 의미를 명료하게 하는 방법 사용
- 사례: 홈페이지 방문자 분석
  - 데이터 분류
    - 방문자 데이터 분류: IP, 방문시간, 접속 페이지, 유입 경로, 웹 브라우저 종류/버전. OS 종류와 해상도, 사용자 ID
    - 매출 데이터 분류: 판매 시간, 고객, 제품, 매장, 관련 부터
    - 콜센터 데이터 분류: 통화 시간, 고객명, 아이디, 문의 종류, 관련 제품, 해결 여부, 해결 방식, 담당자
  - 일정 기준으로 합산: 시간별, 날짜별, 월별
  - 데이터 분석: 챗GPT 에게 물어보기 -> 급등락에 따른 이벤트나 원인 파악하려고 실마리를 줌

### 예상치, 의지치, 목표치, 결과치
- `예상치`: 전제, 근거, 생각을 정량 데이터로 나타낸 것 
- `의지치`: 의지를 보였을 때 다다를 수 있는 숫자
  - 역시 전제, 근거, 생각으로 표현해야 할 것
  - ex) 이벤트를 진행하면 비용 100만원 당 접속자가 250명 늘어날 것으로 예상하지만, 이벤트를 잘 구성하고 배너 디자인을 최신 트렌드에 맞춰서 100명을 추가로 확보하겠음
- 의지치와 목표치가 같지 않을 수 있음

### 통찰과 시사
- PPT에서 왼쪽에 차트를 그리고, 오른쪽에 차트를 설명 -> 굳이 필요없음
- 챗GPT에게 알려주고 `통찰과 시사를 해달라고 요청`
  - ex) 다음 데이터를 보고 통찰이나 시사점을 알려줘. 20년 1분기 5억원, 2분기 6억원, 3분기 7억, 4분기 8억원, 21년 1분기 5억원, 2분기 7억원, 3분기 9억, 4분기 10억원, 22년 1분기 6억원, 2분기 10억원, 3분기 11억, 4분기 12억원
- 통찰(과거, Why Wo): 패턴 + 인과
- 시사(미래, So What): 예측 + 대응

### 데이터 분류와 방안
- 데이터 분류만 잘 해도 통찰과 시사를 얻을 수 있음
   - ex) 다음 데이터는 경비 내역이다. 데이터를 분석해서 경비를 줄이는 방안을 알려줘. 점심: 8000원, 치약: 2000원, 영화: 9000원, 저녁: 18000원, 커피: 3000원, 빗자루: 5000원
- 데이터 분류표 만들기

- 구분합과 비중 계산
  - ex) 구분별 부분합과 비중을 표로 보여줘
